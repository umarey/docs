## GET /api/v1/alerts

Retrieve a paginated list of triggered alerts for your workspace.

Alerts are generated by alert rules configured in the dashboard. Rules can trigger on conditions such as:

- Specific actions (e.g. `user.deleted`)
- Unusual activity volume
- Actions from new IP addresses
- Actions from specific actors
- Custom pattern matches

### Query Parameters

<ParamField query="cursor" type="string">
  Pagination cursor from a previous response.
</ParamField>

<ParamField query="per_page" type="integer" default="25">
  Number of alerts per page (max 100).
</ParamField>

### Alert Delivery

Alerts are delivered via the channels configured on each rule:

- **Email** -- Sent to workspace members with admin or owner roles.
- **Webhook** -- POSTed to your endpoint with an HMAC-SHA256 signature in the `X-Immutable-Signature` header.

### Example

```bash
curl https://getimmutable.dev/api/v1/alerts?per_page=10 \
  -H "Authorization: Bearer imk_your_api_key_here"
```
