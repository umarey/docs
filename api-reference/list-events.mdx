## GET /api/v1/events

Retrieve a paginated list of events with optional filters.

### Query Parameters

<ParamField query="actor_id" type="string">
  Filter by actor identifier.
</ParamField>

<ParamField query="action" type="string">
  Filter by action. Supports wildcard patterns (e.g. `document.*`).
</ParamField>

<ParamField query="resource_id" type="string">
  Filter by primary resource identifier.
</ParamField>

<ParamField query="session_id" type="string">
  Filter by session identifier.
</ParamField>

<ParamField query="target_type" type="string">
  Filter by target type (requires `target_id`).
</ParamField>

<ParamField query="target_id" type="string">
  Filter by target identifier (requires `target_type`).
</ParamField>

<ParamField query="search" type="string">
  Case-insensitive search across actor_name, action, resource_name, and metadata.
</ParamField>

<ParamField query="since" type="string">
  ISO 8601 timestamp. Return events created at or after this time.
</ParamField>

<ParamField query="until" type="string">
  ISO 8601 timestamp. Return events created at or before this time.
</ParamField>

<ParamField query="cursor" type="string">
  Pagination cursor from a previous response.
</ParamField>

<ParamField query="per_page" type="integer" default="25">
  Number of events per page (max 100).
</ParamField>

### Pagination

To iterate through all results, pass the `next_cursor` value from each response as the `cursor` parameter in the next request. When `has_more` is `false`, you have reached the end.

```typescript
let cursor: string | undefined;
do {
  const page = await immutable.events.list({ cursor, per_page: 100 });
  for (const event of page.data) {
    process(event);
  }
  cursor = page.has_more ? page.next_cursor : undefined;
} while (cursor);
```
